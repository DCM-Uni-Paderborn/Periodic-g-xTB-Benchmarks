# CP2K g-xTB `KGROUP_OWNER` qualification

This archive qualifies the CP2K-side ownership precursor for distributed
Brillouin-zone-coupled g-xTB exchange.  It does **not** claim k-group wall-time
scaling of the save_tblite kernel.

## What the selectable path changes

- `CP2K_GXTB_EXCHANGE_STREAM_MODE=KGROUP_OWNER` uses only the official CP2K
  `kp_range`, `kp_dist`, `nkp_groups`, `para_env_kp`, and
  `para_env_inter_kp` metadata.
- Each rank stores only the irreducible densities of its CP2K k-point group.
  For `G` equal groups this changes that CP2K allocation from
  `nAO^2*nspin*nred` to `nAO^2*nspin*(nred/G)` complex values per rank.
- Each full-mesh star member is generated by exactly one owner-group leader.
  A blockwise inter-group reduction checks an integer owner count of exactly
  one before the block enters the provider.
- Exactly one global rank owns the public save_tblite stream.  This avoids
  replicated provider states but deliberately leaves the coupled kernel
  serial, because the public API cannot merge partial k-to-R accumulators.
- Full-mesh Fock blocks return one at a time.  Only the irreducible owner folds
  each response; CP2K then distributes the reduced result over its existing
  leader and intra-group communicators.

The unchanged DBCSR overlap transform still requires every global rank to
enter the same collectives for every full-k block.  Together with the single
provider state, this means the present result is a correctness and ownership
baseline, not a performance result.

## In-process oracle

`CP2K_GXTB_QUALIFICATION_FULLMESH_ORACLE_ITERATION=1` evaluates a dense,
complete-mesh provider oracle in the same SCF iteration.  Each accepted run
prints

```
GXTB-QUALIFICATION_ONLY KGROUP-ORACLE iter=1 dE=... dVsh=... dFfold=...
```

and aborts if any residual exceeds `1e-10`.  `qualification_summary.tsv`
contains the maximum printed residual over the two force-evaluation calls and
the separately parsed final total-energy, force, and stress differences to
the pre-existing full-mesh reference runs.

Coverage includes one, two, and four CP2K k groups; MPI ranks greater than the
number of irreducible k points; unequal star sizes; time reversal; UKS;
SPGLIB and K290 symmetry; a shifted mesh; and 1D, 2D, and 3D periodicity.  All
accepted outputs contain `PROGRAM ENDED`.  The CH4 full-mesh case was also
rebuilt and repeated with the Release executable; its in-process residuals
are identical to the Debug result within printed precision.

The tiny H2 `3x1x1`, four-rank/group-size-two local macOS Debug experiment is
not an accepted result: that executable hit the same small-matrix SIGILL/hang
class seen in the legacy tiny-H2 run.  Four-group behavior is instead covered
by the clean CH4 `2x2x2` run; ranks greater than `Nk` are covered by the clean
CH4 Gamma run.  No result from the failed experiment enters the table.

## Files

- `runs/*_oracle_final/cp2k.out`: accepted raw CP2K outputs.
- `inputs/*.inp`: exact CP2K inputs used by the accepted cases.
- `references/*.out`: frozen pre-existing complete-mesh outputs used for the
  separately parsed energy/force/stress deltas.
- `qualification_summary.tsv`: numerical matrix and explicit coverage.
- `compare.py`: parser used for final energy/force/stress comparisons.
- `verify.py`: re-evaluates every table row from the frozen raw outputs and
  fails on incomplete runs, metadata drift, or numerical-gate violations.
- `provider_partial_accumulator_abi.md`: minimal provider API still required
  for genuine partial k-to-R execution.
- `source_snapshot/`: exact formatted CP2K interface/type sources and audit
  document used by the final Debug and Release builds.
- `independent_review/`: source/communicator/weighting audit plus additional
  Linux Gamma P=4 and time-reversal P=6 cases with `P > nred`.  The audit
  explicitly limits the memory claim to the owner-local CP2K density and
  confirms that the provider kernel is still single-state and unaccelerated.
- `SHA256SUMS`: hashes of the frozen evidence files.
